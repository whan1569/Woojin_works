# 온도제어 시스템 채널 오류 문제 분석 및 개선 보고서

## 1. 개요

본 보고서는 히터 제어 시스템에서 특정 존(Zone[10])만 이상 동작하는 문제에 대해 원인을 분석하고, 코드 리팩토링을 통해 개선한 내용을 정리한 것이다.

---

## 2. 문제 현상

* 특정 존(Zone[10])에서만 “외부장치입력 승온동작 이상” 알람 발생
* 동일 설비 및 배선에서 기존 코드에서는 정상 동작
* 신규 코드 적용 이후 동일 위치에서만 반복적으로 문제 발생
* 다른 존(Zone[9], Zone[11] 등)은 정상

---

## 3. 기존 구조 분석

### 3.1 센서 타입 설정 로직

* 온도카드 정보 문자열 (`ModuleInformation_tempcard`)을 기반으로 파싱
* `tokenST` 값을 통해 카드 타입(ST1, ST11 등) 판별
* ATA492 사용 시 채널 분기 수행

### 3.2 채널 분기 방식 (기존 문제 코드)

```c
if (tokenST != 0 && MA_FIX.ARCONF.UseATA492 == 1)
{
    if (uCounter > 0 && !strcmp(prev, curr))
        Ch02;
    else
        Ch01;
}
```

---

## 4. 문제 원인 분석

### 4.1 ST11 조건 누락

* 기존 정상 코드:

  * `ST11`일 때만 채널 분기 수행
* 변경 코드:

  * 모든 카드에 대해 채널 분기 수행

→ 단일 채널 카드까지 2채널로 잘못 인식

---

### 4.2 채널 판별 방식의 구조적 문제

* 이전 카드와 문자열 비교로 채널 결정
* 순서 의존적 구조

문제점:

* 인덱스 변경 시 오동작
* 동일 문자열 반복 시 잘못된 채널 선택
* 특정 존만 영향 발생 가능

---

### 4.3 문자열 기반 파싱 취약성

* `strtok` 사용
* NULL 체크 부족
* 포맷 의존적 구조

문제점:

* 잘못된 문자열 입력 시 크래시 가능
* 유지보수 어려움

---

### 4.4 중복 코드 및 비일관성

* 전체 변경 로직 / 개별 변경 로직 분리
* 동일 로직이 여러 곳에 중복 존재

문제점:

* 수정 누락 가능성
* 일부 경로만 동작 불일치 발생

---

### 4.5 상태 머신 불명확

* 숫자 기반 상태 처리 (0,1,2)
* 의미 불명확

문제점:

* 흐름 추적 어려움
* 디버깅 비효율

---

## 5. 영향 분석

* Zone[10] (두 번째 채널)만 오류 발생
* 외부장치 알람과 연계되어 오동작
* 실제 하드웨어 이상이 아닌 소프트웨어 로직 문제
* 동일 구조에서 재발 가능성 높음

---

## 6. 개선 내용

### 6.1 채널 분기 로직 개선

#### 변경 전

```c
문자열 비교 기반 (prev == curr)
```

#### 변경 후

```c
index 기반 채널 결정
index % 2
```

효과:

* 순서 의존 제거
* 안정적 채널 매핑

---

### 6.2 ST11 조건 복구

```c
if (tokenST != 0 && !strcmp(tokenST, \"ST11\"))
```

효과:

* 잘못된 채널 분기 방지
* 카드 타입별 정확한 처리

---

### 6.3 파싱 로직 분리

```c
GetTokenST()
```

효과:

* 안정성 확보
* NULL 처리 명확화

---

### 6.4 공통 함수화

```c
SetSensorType()
```

효과:

* 중복 제거
* 유지보수 용이

---

### 6.5 상태 머신 개선

```c
STEP_INIT / STEP_WRITE / STEP_DONE
```

효과:

* 가독성 향상
* 흐름 명확화

---

### 6.6 구조 단순화

* 불필요한 주석 제거
* 로직 흐름 직선화

---

## 7. 개선 결과

| 항목    | 개선 전   | 개선 후       |
| ----- | ------ | ---------- |
| 채널 분기 | 문자열 비교 | 인덱스 기반     |
| 카드 조건 | 없음     | ST11 조건 적용 |
| 안정성   | 낮음     | 높음         |

## 8. 결론

본 문제는 하드웨어 이상이 아닌 **채널 분기 로직의 구조적 결함**으로 인해 발생하였다.

특히:

* ST11 조건 누락
* 문자열 기반 채널 판별

이 두 가지가 결합되어 특정 존(Zone[10])에서만 오류를 유발하였다.

리팩토링을 통해:

* 채널 결정 방식 개선
* 로직 단순화 및 안정화

를 수행하였으며, 동일 유형의 문제 재발 가능성을 제거하였다.

---

## 9. 향후 개선 권장사항

* Zone ↔ 외부입력 매핑 테이블 명시화
* 로그 시스템 추가 (에러 추적용)
* 문자열 기반 구조 → 구조체 기반 전환
* 센서/카드 설정 모듈 분리
